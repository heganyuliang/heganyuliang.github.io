小白浅谈Xposed与Magisk

前一篇文章说要介绍 Magisk，想了想干脆把 Xposed 一起介绍了吧。

Xposed 无疑是站长接触安卓以来见过的最强力的东西，曾经为了使用它而坚决不上 7.0（虽然后来还是叛变革命了）。Xposed 框架由 xda 大神 rovo89 开发，通过替换安卓系统 /system/bin 目录下的文件，实现对系统某些功能的接管，进而给予基于 Xposed 框架开发的 app（Xposed 模块）更多权限。用户可以通过安装模块深度定制自己的安卓系统，实现诸如 qq、微信防撤回、为单独应用修改 dpi 或语言、强行阻止毒瘤常驻后台等功能。具体的实现原理萌新站长也不是很明白……感兴趣的读者还请自行搜索，反正我是没看懂（懵逼.jpg）。

刷入 Xposed 的过程与之前介绍的刷入 Magisk 类似，都是先在第三方 rec 刷入 zip 包（卡刷，区别于刷第三方rec时的线刷）再安装启动器，不过需要注意的是 Xposed 框架不自带 root，需要先 root 后再刷。既然 Xposed 本身就会修改 /system，因此选择 root 也无须选择 systemless 模式，使用 SuperSu就好，最新版的 xda 链接在此，如果对被收购后的版本不放心，也可以选择被收购前的最后一版 v2.76。

当然，这种深度介入系统的行为必然伴随着风险。装上Xposed框架后有时会面临着无法开机的情况，这个时候就需要卡刷卸载包了。话不多说，安装包和卸载包在这里下载：

https://dl-xda.xposed.info/framework/

注意，下载时请选择对应的版本，安卓 5.0 是 SDK21，5.1 是 SDK22，6.0 是 SDK23，7.0 是 SDK24，7.1 是 SDK25，8.0 是 SDK26，8.1 是 SDK27， Xposed 暂不支持 8.0 及以上的版本。另外，还需根据自己手机的 CPU 选择正确的 arm 或 x86 版本，若不清楚请自行搜索或在评论里询问站长。卸载包不分安卓版本，只要选择正确的 CPU 版本就好。

不愿意修改 /system 文件夹的玩家怎么办呢？Magisk 就在这时横空出世了。通过站长还是看不明白的原理，Magisk 能实现在不修改 /system 文件夹的情况下深度定制系统的效果。并且，Magisk 自带开源的 root，不需要再去特意 root。并且，rovo89 配合 Magisk 作者 topjohnwu 开发了基于 Magisk 的 Xposed 框架，应该是能支持所有的 Xposed 模块，有兴趣的玩家可以自己尝试。

Magisk 的 xda 链接在此，上一篇文章里也有介绍，这里就不废话了。

总之，Magisk 相对来说上手难度较低，也比较安全，建议有兴趣的读者先尝试一下这个玩法。不过 Xposed 的可玩性目前来说是无人能及的，有时间精力的朋友可以大胆地尝试，你会发现一个神奇的新世界。

最后，Xposed Manager 和 Magisk Manager 酷安都有下载，这里就不贴地址了。
